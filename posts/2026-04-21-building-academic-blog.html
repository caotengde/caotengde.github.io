Building an Academic Blog with GitHub Pages

A personal academic blog does not need to be complicated. The important parts are stable access, clear structure, and a place where papers, projects and notes can accumulate naturally.

What This Site Contains

This version uses plain static files and does not require a build step. The homepage includes a short profile, research interests, projects, publications, recent notes, an academic timeline and contact links. The notes page collects longer research notes.

Maintenance Pattern

When adding a publication, keep the title, authors, year, venue and links easy to verify. For notes, it is usually better to focus on one question at a time and record the motivation, materials, method, conclusions and open questions.

Deployment

After these files are pushed to the GitHub repository named caotengde.github.io, GitHub Pages publishes the site at https://caotengde.github.io.
